## Bug Reporting Process

### How to Report
1. Check existing bugs for duplicates
2. Use bug template format
3. Include reproduction steps
4. Assign appropriate priority
5. Set target fix date

### Priority Guidelines
- **P0** — System down, data loss, security vulnerability
- **P1** — Major feature broken, significant user impact
- **P2** — Minor feature issues, workaround available
- **P3** — Cosmetic issues, enhancement requests
